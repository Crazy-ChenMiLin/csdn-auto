**主题**: Kafka 灾难回放

---

## 📚 学习目标

理解 Kafka 在灾难场景下的数据恢复和回放机制。

---

## 🔑 核心知识点

### 1. Kafka 消息持久化
- Kafka 消息存储在磁盘上，支持持久化
- 消息保留策略：基于时间或大小
- 日志段文件（Log Segment）管理

### 2. 消费者 Offset 管理
- **消费位移（Offset）**: 记录消费者读取的位置
- Offset 存储位置：
  - Kafka 内部主题 `__consumer_offsets`
  - 或外部存储（如数据库）
- Offset 提交方式：自动提交 vs 手动提交

### 3. 灾难回放机制
- **重置 Offset**: 将消费者位移回退到指定位置
- **从头消费**: `auto.offset.reset = earliest`
- **按时间回放**: 根据消息时间戳重新消费
- **按分区回放**: 针对特定分区进行数据恢复

### 4. 关键配置参数
```properties
# 从最早的消息开始消费
auto.offset.reset=earliest

# 手动提交 offset
enable.auto.commit=false

# 消息保留时间（灾难恢复窗口）
log.retention.hours=168

# 消息保留大小
log.retention.bytes=1073741824
```

### 5. 灾难恢复策略
1. **数据备份**: 定期备份 Kafka 数据和元数据
2. **MirrorMaker**: 跨集群数据复制
3. **Offset 重置**: 使用 `kafka-consumer-groups` 命令重置位移
4. **时间戳回放**: 根据业务时间点重新消费数据

---

## 🛠️ 实践命令

### 查看消费者组状态
```bash
kafka-consumer-groups.sh --bootstrap-server localhost:9092 \
  --describe --group <group-id>
```

### 重置消费者 Offset
```bash
# 重置到最早
kafka-consumer-groups.sh --bootstrap-server localhost:9092 \
  --group <group-id> --reset-offsets --to-earliest --execute

# 重置到指定时间
kafka-consumer-groups.sh --bootstrap-server localhost:9092 \
  --group <group-id> --reset-offsets --to-datetime 2024-01-01T00:00:00 --execute
```
